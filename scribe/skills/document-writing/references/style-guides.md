# Document Style Guides

Writing style conventions for different document types and audiences.

---

## Tone Matrix

| Tone | Vocabulary | Sentence Length | Structure | Example Use |
|------|-----------|-----------------|-----------|-------------|
| `formal` | Professional, domain-specific | Medium to long | Structured paragraphs, numbered lists | Board reports, contracts |
| `casual` | Conversational, accessible | Short to medium | Bullet points, informal headings | Internal memos, team updates |
| `technical` | Precise, specialized terms | Variable | Code blocks, specifications, tables | API docs, technical proposals |
| `executive` | High-level, outcome-focused | Short, punchy | Key metrics, recommendations, TL;DR | Executive summaries, dashboards |

---

## Template Conventions

### Report
- Numbered headings (1., 1.1, 1.2)
- Executive summary at top
- Data tables and charts where applicable
- Conclusions and recommendations at end
- Formal or executive tone

### Memo
- **To/From/Date/Subject** header
- Brief, one-page preferred
- Action items highlighted
- Casual or formal tone

### Proposal
- Problem statement → Proposed solution → Benefits → Timeline → Cost
- Persuasive structure
- Supporting data and evidence
- Formal tone

### Manual
- Task-oriented structure
- Step-by-step procedures with numbered lists
- Warnings, notes, and tips in callout blocks
- Technical tone with accessible explanations
- Glossary of terms

### Slides (PPTX)
- One key idea per slide
- 6 words per bullet, 6 bullets per slide (maximum)
- Visual-first: charts, diagrams, images over text
- Speaker notes for detailed content
- Executive or casual tone

### Spreadsheet (XLSX)
- One topic per worksheet
- Header row with bold formatting
- Data validation where applicable
- Summary/dashboard sheet at front
- Cell comments for context

## Language-Specific Conventions

### Japanese (ja)
- Use desu/masu form for formal tone
- Prefer kanji with furigana for technical terms on first use
- Section numbering: standard Arabic numerals
- Avoid excessive katakana for translatable concepts
- Honorific language (keigo) for executive audience

### English (en)
- Active voice preferred over passive
- Oxford comma in lists
- Abbreviations spelled out on first use
- American English spelling (unless British English specified)

## Formatting Rules

### Headings
- Use `#` hierarchy consistently (no skipping levels)
- Keep headings under 60 characters
- Use sentence case (capitalize first word only)

### Lists
- Use `-` for unordered lists
- Use `1.` for ordered/procedural lists
- Maximum 2 levels of nesting

### Tables
- Include header row
- Align numeric columns right
- Keep tables under 6 columns where possible
- Add a brief caption above the table

### Code/Data Blocks
- Use fenced code blocks with language identifier
- Keep blocks under 30 lines (split longer blocks)

### Cross-References
- Use `[→ See §N]` placeholder format during parallel writing
- Lead agent resolves to actual links during merge
